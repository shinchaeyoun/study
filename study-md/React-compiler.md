# React 컴파일러


**React 컴파일러는 React 19에서 새롭게 도입된 기능으로, 개발자가 직접 `useMemo`, `useCallback`, `React.memo` 같은 최적화 코드를 작성하지 않아도 자동으로 성능 최적화를 해주는 도구입니다.** 즉, 불필요한 리렌더링을 줄이고 애플리케이션을 더 빠르고 효율적으로 실행할 수 있게 도와줍니다.  

---

## 🔑 React 컴파일러 핵심 개념
- **자동 메모이제이션**: 기존에는 성능 최적화를 위해 수동으로 `useMemo`나 `useCallback`을 작성해야 했지만, 이제 컴파일러가 알아서 처리합니다.
- **리렌더링 제어**: 컴포넌트가 불필요하게 다시 그려지는 상황을 줄여줍니다.
- **점진적 도입 가능**: 프로젝트 전체에 바로 적용하지 않고, 일부 컴포넌트부터 단계적으로 적용할 수 있습니다.
- **개발자 경험 개선**: 최적화 코드를 덜 작성해도 되므로 코드가 간결해지고 유지보수가 쉬워집니다.

---

## ⚖️ 비교: 기존 방식 vs React 컴파일러

| 구분 | 기존 React (React 18 이하) | React 19 + Compiler |
|------|---------------------------|----------------------|
| 성능 최적화 | 수동으로 `useMemo`, `useCallback`, `React.memo` 작성 | 자동 최적화 (컴파일러가 처리) |
| 코드 복잡도 | 최적화 코드가 많아져 복잡 | 코드 간결, 유지보수 용이 |
| 도입 방식 | 개발자가 직접 최적화 전략 설계 | 점진적 도입 가능, 자동 적용 |
| 디버깅 | 최적화 누락 시 성능 저하 발생 | 컴파일러가 최적화, 디버깅 가이드 제공 |

---

## 🌟 왜 중요한가?
React는 오랫동안 **강력한 UI 라이브러리**였지만, Vue나 Svelte처럼 전용 컴파일러가 없다는 점이 약점으로 지적되었습니다. React 19의 컴파일러는 이 한계를 극복하며, **개발자가 성능 최적화에 쏟는 시간을 줄이고 더 직관적인 코드 작성**을 가능하게 합니다.

---

👉 요약하면, **React 컴파일러는 "자동 성능 최적화 엔진"으로**으로, 개발자가 최적화 코드를 직접 작성하지 않아도 React 앱을 더 빠르고 효율적으로 만들어주는 혁신적인 기능입니다.
